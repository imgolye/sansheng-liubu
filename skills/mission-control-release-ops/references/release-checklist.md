# Release Checklist

Use this checklist before cutting a release:

1. Confirm version markers are aligned across install scripts, UI runtime constants, README highlights, and changelog.
2. Run syntax or compile checks for touched Python files.
3. Run feature-specific validation for new product surfaces and admin actions.
4. Verify packaging outputs exist and contain the expected root files.
5. Summarize any unverified risk, especially UI smoke tests or live channel integrations.
